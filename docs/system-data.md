# System Data とは何か（概要）

macOSのストレージ表示にある「System Data（システムデータ）」は、Appleの定義では **他のカテゴリに属さない Apple/サードパーティのファイルをまとめた一般カテゴリ** です。

そのため中身は環境や時期で変動し、「System Dataをゼロにする」ような危険な最適化は推奨されません。

## macdiet の方針

- macOS標準UIのカテゴリ表示を“正確に再現”することは目的にしません（分類はブラックボックス寄りで変動するため）。
- 開発者環境で頻出の肥大化要因（Xcode派生物、Simulator、Docker、ログ/キャッシュ等）を、再現可能な根拠（Evidence）付きで優先的に推定します。
- 既定はR0（Read-only）。実行はホワイトリスト化された安全な範囲に限定し、原則 `~/.Trash` への移動とします。

## 権限と「未観測」

Full Disk Access がない場合、走査できない領域が発生する可能性があります。

より正確にするには、macdiet を実行しているターミナルに Full Disk Access を許可してください（システム設定 → プライバシーとセキュリティ → フルディスクアクセス）。

macdiet は「見えない領域」を 0 と断定せず、可能な範囲で「未観測」として表示し、必要に応じて権限確認のヒントを提示します（断定はしません）。

また、ディレクトリ走査中のエラー（例: 権限不足）を検出できた場合は、`errors=` から **未観測バイト（unobserved_bytes）を概算** して表示することがあります（参考値）。
